# where2go-weekend 项目宪法

## 1. 项目定义

where2go-weekend 是基于 POI 快照、历史比较和外部证据，观察品牌门店及城市街区变化的数据项目。

当前阶段优先打牢可信的数据、证据和分析基础。

项目不是：
- 实时地图搜索助手
- 打开页面时临时调用高德 API 的地点推荐系统
- 大众点评式排行榜或评分平台
- 地图和报告是数据产物的使用层，不是数据事实来源

## 2. 三个核心原则

### 原则一：采集与使用严格分离

- 只有明确的采集命令可以调用高德或其他外部数据 API。
- analyze、compare、map、slice、report 等读命令只能读取已落盘数据。
- 生成地图、报告或重新分析历史结果时，不得隐式触发采集。
- 相同输入应能稳定复现相同分析结果。
- 展示层不得重新引入采集依赖。

这一原则直接影响架构：`run.py` 的采集集中在 step 2，地图和报告在 step 5/6 基于已落盘 CSV 生成，不调 API。品牌 POI 的 `scan_to_snapshot` 也只负责一次集中采集，后续 map/compare 命令独立运行。

### 原则二：只对变化候选执行高成本验证

- 全量存量数据只进行低成本扫描、标准化、匹配和比较。
- 联网搜索、证据收集及复杂判断只用于变化候选。
- 禁止对全部存量门店无差别联网搜索。
- 优先缩小候选集，而不是降低最终验证标准。
- 已有本地数据、证据和缓存可回答时，不重复搜索。

store-opening-watch 的候选分级策略是这一原则的具体实现：只对高优先级变化候选启动外部搜索和证据验证，其余候选仅记录或进入低成本处理流程。具体等级、事件类型和处理规则以该 Skill 的 references 为准。

### 原则三：事实、推断与建议分离

明确区分：

- **Observation（观测）**：数据中直接观测到的内容。例如"本期快照中淮海路新增 1 条 Nike POI"。
- **Inference（推断）**：通过规则和证据形成的判断。例如"新增 POI 无历史匹配，列为新增候选，置信度中"。
- **Recommendation（建议）**：面向用户的解释和行动建议。例如"建议实地走访确认当前是否已开业"。

要求：
- 建议必须能够追溯到推断，推断必须能够追溯到观测与证据。
- 三种内容不得在数据字段或报告语言中混写。
- 评分只能作为分析工具，不能代替事实和解释。

## 3. 数据可信原则

- 数据缺失不等于没有变化。
- POI 新增不直接等于真实新开业。
- POI 消失不直接等于真实关闭。
- 名称变化、地址修正、楼层变化、坐标漂移和数据源更新必须与真实门店事件区分。
- 快照属于不可变原始观测，后续分析不得覆盖历史快照。
- 未达到确认标准的事件必须标记为疑似、待验证或无法判断。
- 禁止将低置信度推断写成确定事实。
- 外部证据必须记录来源和时间。

## 4. 项目产品视角

where2go 不只观察哪里已经最热门，更关注哪里正在发生变化。

门店是城市变化的信号，不是城市本身的全部。观察对象不仅是单店，也包括街区、园区、商业空间和业态结构。

数据帮助发现变化，但不能替代真实体验。对城市保持好奇，但不把噪声包装成趋势。

最终输出应回答：发生了什么、为什么重要、用户可以做什么。

不使用缺少事实或证据支持的夸张表达，不以流量化措辞替代具体判断。评分只能作为分析工具，不替代具体事实。

对外报告语言应保持"POI 新增""首次观测""疑似""待验证"等精确表述。在面向用户的解释层中，可以转化为"这个街区最近出现了新空间""建议周末去观察"等行动建议，但转化过程必须可追溯。

## 5. Skill 路由

- 品牌 POI 采集、快照和地图流程使用 `brand-map-pipeline`。
- 门店新增、关闭、迁址、改名及证据验证使用 `store-opening-watch`。
- 街区 POI 采集与周末变化分析（原始 run.py 工作流）不依赖特定 Skill，直接遵循本宪法。
- 本宪法只写共同原则，不复制具体 Skill 的完整 SOP。具体任务执行时按需加载对应 Skill。
- Skill 与本宪法冲突时，以本宪法的硬约束为准。

品牌 POI 任务和门店变化监控任务可能出现在同一对话中，此时应分别加载两个 Skill，各自遵循其流程，但统一受本宪法约束。例如：先按 brand-map-pipeline 采集快照，再按 store-opening-watch 比较两期快照识别变化。

## 6. 全局规则继承

- 跨项目工作人格来自全局 `~/.config/opencode/SOUL.md`，由全局 `instructions` 加载。
- Context7 使用步骤、工具使用约束和 zihao raccoon 视觉规范继承全局 `~/.config/opencode/AGENTS.md`。
- 本项目不重复完整的全局视觉规范。HTML、地图、报告继承 zihao raccoon 体系，同时保持 where2go 的信息层级和城市观察视角。
- 项目特有视觉约束：地图配色优先使用品牌配置中的 `marker_color`；报告标题使用 `--zh-blue: #174A7C`。

## 7. 禁止事项

- 禁止在读命令中调用采集 API。
- 禁止覆盖历史快照来修正分析结果。
- 禁止把疑似变化写成确认事件。
- 禁止全量无差别执行高成本搜索。
- 禁止没有来源地生成真实门店事实。
- 禁止把采集逻辑重新耦合进地图和报告。
- 禁止为了演示效果制造数据或结论。

违反以上禁止事项即视为项目宪法违规，Agent 应在任何后续响应中主动修正，而非继续执行基于违规数据或方法得到的结果。
