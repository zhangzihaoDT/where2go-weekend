# Source Strategy：来源策略与搜索方法

> 何时读取：需要对变化候选搜索外部开业证据时。

---

## 1. 核心原则

只对 P0/P1 变化候选执行搜索。每次对单个候选门店生成 3–6 个精确查询，不做无边界泛搜。

---

## 2. 查询模板

推荐查询模板：

```text
{品牌} {商场} 开业
{品牌} {商场} 正式开业
{品牌} {商场} 试营业
{门店完整名称} 开业
{品牌} {城市} {商场} 新店
{商场} {品牌} 入驻
```

必要时补充：

```text
site:品牌官方域名 {商场}
site:商场官方域名 {品牌}
微信公众号关键词检索
品牌官方微博或小红书账号检索
```

不得只搜索品牌名或"新店开业"这类宽泛关键词。

---

## 3. 来源优先级

| 来源 | 权重 | 说明 |
|---|---|---|
| 品牌官方公告（官网/公众号/微博） | 最高 | 可直接确认开业日期 |
| 商场官方公告 | 高 | 入驻信息可靠 |
| 正式新闻报道 | 高 | 有明确时间、地点 |
| 品牌官方门店列表 | 中 | 可佐证已开业，但通常无日期 |
| 商场招商/招聘信息 | 中 | 间接信号 |
| 用户生成内容（小红书/大众点评） | 中 | 需交叉验证 |
| 地图 POI（新增） | 低 | 不能单独确认开业 |
| 无来源聚合页面 | 低 | 不可核验 |

---

## 4. 搜索停止条件

满足以下任意一条即可停止搜索：

- 找到高置信度证据
- 找到至少两个相互独立的中置信度证据
- 已搜索 3–6 个精确查询且均无有效结果

---

## 5. 缓存与重复搜索规则

- 每次搜索必须记录查询词、来源、日期和结果
- 同一候选门店不重复搜索（除非有新证据或用户要求刷新）
- 已确认开业的历史事件不重复检索
- 搜索结果缓存应包含搜索日期范围

---

## 6. 成本保护

- 不得对全部 400+ 家存量门店反复进行无差别网页搜索
- 不得为了提高"命中率"放宽到无法解释的空间匹配
- 人工复核队列应控制在少量高价值候选
- 不为补齐全部历史开业日期而无边界回溯
