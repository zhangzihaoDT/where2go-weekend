# 案例：迁址与更名 — 同一商场楼层变化

> 展示非典型变化事件的判断逻辑。

---

## 案例 A：同一商场楼层冲突

### 输入观测

同一品牌、同一商场出现两条记录：

| 字段 | 当前 POI A | 当前 POI B |
|---|---|---|
| 名称 | 阿迪达斯(XX商场6F店) | 阿迪达斯(XX商场8F店) |
| 品牌 | Adidas | Adidas |
| 商场 | XX商场 | XX商场 |
| 楼层 | 6F | 8F |
| 电话 | 相同 | 相同 |

### 判断过程

- 同品牌 ✓
- 同商场 ✓
- 同电话 ✓
- 不同楼层
- 店型相同

### 最终状态

```text
event_type: possible_floor_change
confidence: medium
```

不得直接判断为两家门店，也不得直接判断一家关闭。需后续复核或下期快照确认。

---

## 案例 B：同一商场重复 POI

### 输入观测

同一品牌、同一商场、相似名称出现两条 POI：

```text
耐克Nike(上海月星环球港店)
中山北路3300号上海月星环球港L2层L2010
```
以及：
```text
耐克Nike(上海月星环球港店)
中山北路3300号上海月星环球港L2层
```

### 判断过程

- 同品牌 ✓
- 同商场 ✓
- 同名称 ✓
- 同楼层 ✓
- 同电话（假设相同）

### 最终状态

```text
event_type: possible_duplicate
action: 合并为同一门店实体，保留两条来源记录
```

不形成新开或关闭事件。

---

## 案例 C：迁址（跨商场）

### 输入观测

| 字段 | 上期 | 本期 |
|---|---|---|
| 名称 | 品牌X(老商场店) | 品牌X(新商场店) |
| 地址 | 老商场 | 新商场 |
| 电话 | 021-XXXX | 021-XXXX（相同） |
| 旧 POI | 存在 | 消失 |

### 判断过程

- 同电话 ✓
- 距离变化 > 1km ✓
- 名称变化（商场名更新）

### 最终状态

```text
event_type: relocated_store
confidence: high（基于同电话确认）
```

旧 POI 消失不生成 `possible_closed_store`，它和新增是同一次迁址事件的两面。
