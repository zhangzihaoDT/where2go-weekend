## 核心原则：先扫描，再可视化

所有涉及高德 POI API 调用的地图生成脚本必须遵守 **两阶段分离** 原则：

### 阶段一：数据采集（API 调用）

- 完成所有品牌的 POI 搜索请求
- 覆盖所有城市、所有关键词、所有分页
- 数据去重、整理
- 此阶段不生成任何 HTML/地图

### 阶段二：可视化（零 API）

- 使用阶段一采集的完整数据
- 生成 HTML 地图
- 不再调任何高德 API
- 所有筛选、交互、样式调整都基于已有数据

### 禁止行为

- ❌ 不允许在用户交互过程中触发新的高德 API 请求
- ❌ 不允许前端筛选/过滤时回源调用 API
- ❌ 不允许为了地图 UI 效果而浪费 API 配额

### 检查清单

- [ ] 所有 API 调用是否在 `main()` 中集中完成？
- [ ] HTML 生成函数是否只读地使用传入数据？
- [ ] 地图筛选是否基于本地数据（JS 数组），而非远程请求？
- [ ] 多次运行是否只有第一次消耗 API 配额？
